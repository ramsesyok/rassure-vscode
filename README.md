# Rassure — VS Code 拡張機能

ローカルのフォルダに JSON ファイルとしてチケットを保存する、サーバーレス・オフライン完結の課題管理 VS Code 拡張です。  
Electron 版 [rassure](https://github.com/ramsesyok/rassure) を VS Code Webview として移植しました。

## 特徴

- **完全オフライン動作** — インターネット接続不要。クローズドネットワーク環境でも動作します
- **データベース不要** — チケットは 1 件 1 ファイルの JSON で保存。Git 管理やネットワーク共有フォルダにも対応
- **VS Code ネイティブ UI** — エディタのテーマに合わせた Webview パネルで表示
- **ボード表示** — チケット一覧をテーブル形式で俯瞰
- **詳細表示** — 行クリックで別タブに詳細・コメントを表示

## 使い方

### 1. フォルダを開く

チケットを保存したいフォルダを VS Code で開きます（`ファイル > フォルダーを開く`）。  
開いているフォルダが自動的にチケット保存先として使われます。別のフォルダを使いたい場合はボード画面の設定から変更できます。

### 2. ボードを開く

コマンドパレット（`Ctrl+Shift+P` / `Cmd+Shift+P`）から以下を実行します。

```
Rassure: ボードを開く
```

### 3. チケットを管理する

| 操作 | 方法 |
|------|------|
| 新規チケット作成 | ボード右上の「新規」ボタン |
| チケット詳細を開く | 一覧の行をクリック |
| チケットを編集する | 詳細画面の「編集」ボタン |
| コメントを追加する | 詳細画面のコメント入力欄 |
| 保存フォルダを変更する | ボード右上の「設定」ボタン |

## チケットデータ形式

チケットは保存フォルダに `T-001.json` のような形式で保存されます。

```json
{
  "id": "T-001",
  "description": "ログイン画面のボタンラベルに誤字あり",
  "target": "login.tsx",
  "category": "誤記",
  "status": "open",
  "priority": "medium",
  "assignee": "yamada",
  "dueDate": "2026-04-30",
  "reporter": "tanaka",
  "createdAt": "2026-04-12T09:00:00.000Z",
  "updatedAt": "2026-04-12T09:00:00.000Z",
  "comments": []
}
```

### ステータス

| 値 | 表示 |
|----|------|
| `open` | オープン |
| `in_progress` | 対応中 |
| `resolved` | 解決済み |
| `closed` | クローズ |

### 優先度

| 値 | 表示 |
|----|------|
| `high` | 高 |
| `medium` | 中 |
| `low` | 低 |

## 指摘種別のカスタマイズ

チケット保存フォルダに `指摘種別` というファイルを置くと、チケット作成・編集時の「指摘種別」選択肢をカスタマイズできます。  
ファイルが存在しない場合は、ボードを開いた際に以下の内容で自動生成されます。

```
誤記
記載不足
要確認
修正依頼
質問
```

1 行 1 カテゴリの形式でテキストエディタから自由に編集できます。

## 開発・ビルド

### 必要環境

- Node.js 18 以上
- VS Code 1.85 以上

### セットアップ

```bash
npm install
npm run build
```

### デバッグ実行

VS Code で `F5` を押すと Extension Development Host が起動します。

### ビルドスクリプト

| コマンド | 内容 |
|----------|------|
| `npm run compile` | Extension Host（TypeScript → CommonJS） |
| `npm run build:webview` | Webview（React+MUI → Vite バンドル） |
| `npm run build` | 上記両方を実行 |
| `npm run watch` | Extension Host の TypeScript 監視コンパイル |

### VSIX パッケージ作成

```bash
npx vsce package
```
